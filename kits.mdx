## Why use a kit system?

It's important to have a kit system in place to give Soup FFA more variety, replayability, and replay value. This is a gameplay system which lets players choose different playstyles while keeping the server balanced, competitive, and fun.

## How does the kit system work?

The kit system lets staff create, edit, enable, disable, and manage kits directly in-game. Each kit can have its own icon, inventory contents, armor contents, potion effects, cooldown, lore, enabled status, and allowed users.

## Configuring kits

<Warning>
  stew.gg kits are actively balanced and may change over time. Kit abilities, cooldowns, armor contents, inventory contents, potion effects, lore, and allowed-user access should be tested before release.
</Warning>

### Creating and managing kits

Staff can create, delete, rename, and list kits using the kit management commands.

<CodeGroup>

```javascript Syntax
  /kit create <name>
  /kit delete <name>
  /kit rename <oldName> <newName>
  /kit list
```

```javascript Example
  /kit create Stomper
  /kit delete Stomper
  /kit rename Stomper StomperV2
  /kit list
```

</CodeGroup>

### Setting the kit icon

Set the icon of a kit to the item currently held in your hand.

<CodeGroup>

```javascript Syntax

```

```javascript Example

```

</CodeGroup>

### Setting kit contents

Set the inventory contents of a kit using your current inventory.

<Warning>
  This is a dangerous configuration command. Incorrect inventory contents may create balance issues or give players unintended items.
</Warning>

<CodeGroup>

```javascript Syntax
  /kit setcontents <name>
```

```javascript Example
  /kit setcontents Stomper
```

</CodeGroup>

## Managing kit access

### Allowing a player to use a kit

Allow a specific player to use a kit.

<Warning>
  This command should only be used when a player is meant to receive access to a restricted kit.
</Warning>

<CodeGroup>

```javascript Syntax
  /kit addallowedplayer <name> <player>
```

```javascript Example
  /kit addallowedplayer Stomper Steve
```

</CodeGroup>

<br />

<Frame>
  ![Add Allowed Player Preview](./images/kits/add-allowed-player.png)
</Frame>

### Removing an allowed player

Remove an allowed player from a kit.

<CodeGroup>


</CodeGroup>

## Editing kit lore

### Adding lore lines

Add a lore line to a kit. Lore is used to describe the kit in menus, selectors, and previews.

<CodeGroup>

```javascript Syntax
  /kit addloreline <kit> <line>
```

```javascript Example
  /kit addloreline Stomper Leap from above and crush nearby enemies.
```

</CodeGroup>

<br />

<Frame>
  ![Add Lore Line Preview](./images/kits/add-lore-line.png)
</Frame>

### Removing lore lines

Remove a lore line from a kit by its index.

<CodeGroup>

```javascript Syntax
  /kit removeloreline <kit> <index>
```

```javascript Example
  /kit removeloreline Stomper 1
```

</CodeGroup>

<br />
